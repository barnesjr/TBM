# TBM Assessment Tool

## Overview

Technology Business Management (TBM) maturity assessment tool. FastAPI backend serves a React SPA and generates export deliverables (DOCX, XLSX, PPTX, PNG). PyInstaller packages everything into a standalone executable for end-user distribution.

## Tech Stack

- **Backend:** Python 3.12+, FastAPI, Pydantic 2, Uvicorn
- **Frontend:** React 19, TypeScript, Vite, Tailwind CSS 4, Recharts, React Router
- **Exports:** openpyxl (XLSX), docxtpl (DOCX), python-pptx (PPTX), matplotlib (radar charts)
- **Packaging:** PyInstaller (macOS ARM + Windows)

## Project Structure

```
frontend/          # React + TypeScript + Vite app
backend/           # FastAPI app (main.py, models.py, data_manager.py, export_engine.py)
backend/static/    # Compiled frontend output (generated by vite build)
framework/         # assessment-framework.json (TBM Council Framework, read-only)
templates/         # DOCX/XLSX/PPTX templates for export engine
exports/           # Generated deliverables output directory
build.py           # Build orchestration (dev, frontend, dist, PyInstaller)
assessment-tool-macos.spec    # PyInstaller spec for macOS ARM
assessment-tool-windows.spec  # PyInstaller spec for Windows
```

## Common Commands

```bash
# Development (starts backend :8751 + frontend :5173 with HMR)
python build.py --dev

# Build frontend only → backend/static/
python build.py --frontend

# Build PyInstaller executable
python build.py

# Create distribution ZIP
python build.py --dist

# Type check frontend
cd frontend && npx tsc --noEmit

# Lint frontend
cd frontend && npm run lint

# Install dependencies
pip install -r requirements.txt
cd frontend && npm ci
```

## API Routes

- `GET /api/health` — health check
- `GET /api/assessment` — load assessment data
- `PUT /api/assessment` — save assessment data
- `GET /api/framework` — load TBM framework definition
- `POST /api/export/{type}` — generate deliverable (findings, executive-summary, gap-analysis, workbook, outbrief, heatmap, quick-wins, cost-transparency-roadmap, all)

## CI/CD

- **ci.yml** — runs on push/PR to main: TypeScript type check (`tsc --noEmit`)
- **release.yml** — triggered by `v*` tags: builds macOS ARM + Windows executables, creates GitHub Release with ZIPs

## Release Process

1. Commit and push changes to main
2. Create and push a tag: `git tag v1.0.x && git push origin v1.0.x`
3. release.yml builds both platforms and creates a GitHub Release

## Important Patterns

### PyInstaller Import Compatibility
Backend modules use try/except import fallbacks because PyInstaller runs `main.py` as `__main__`, breaking relative imports:
```python
try:
    from .models import Foo
except ImportError:
    from models import Foo
```
Any new backend module with relative imports must follow this pattern.

### Frontend Build Output
Vite builds to `backend/static/`. The FastAPI app serves this as a SPA with index.html fallback.

### Data Persistence
- Assessment data: `data.json` (atomic writes with `.bak` backup)
- Framework: `framework/assessment-framework.json` (read-only, ~500+ items across 12 disciplines)

### Frontend State
- Context-based store (`useStore` hook) with 300ms debounce auto-save
- Path alias: `@/*` → `frontend/src/*`
